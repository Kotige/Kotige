## 🔭 De Onde Venho — e Para Onde Vou

Tenho formação em **Física** com ênfase em **Física Computacional** pela **UFSJ** e mestre em **Física** pela UFJF e tenho experiência como **Professor de Física**. Ao longo da minha trajetória, aprendi que física não é apenas uma disciplina e sim uma **forma de pensar**, de formular hipóteses, modelar sistemas complexos, medir incertezas e extrair conclusões confiáveis de dados ruidosos.

Um dia percebi que estava fazendo isso o tempo todo — e que havia uma área inteira do mercado de trabalho que precisava exatamente dessas habilidades: **Ciência de Dados**.


Hoje, além da base científica sólida, carrego experiência prática em **desenvolvimento web** e estudo Ciência de Dados por conta própria, focando em usar das minhas habilidades como Físico integradas à área de Dados e assim construindo projetos de dados prontos para produção com essa combinação única.

## ⚛️ Por Que Física é um Diferencial em Dados?

| Habilidade de Físico | Aplicação Direta em Dados |
|---|---|
| 📐 **Modelagem matemática** | Constrói modelos preditivos com fundamento sólido, não "caixa preta" |
| 📊 **Análise de incertezas** | Interpreta intervalos de confiança e p-valores com rigor real |
| 🔬 **Método científico** | Formula hipóteses testáveis antes de qualquer análise (A/B Testing, inferência) |
| 📉 **Ajuste de curvas e regressão** | Já aplicava regressões e mínimos quadrados muito antes de aprender Scikit-Learn |
| 🧮 **Álgebra linear e cálculo** | Compreende o que acontece *dentro* dos algoritmos de ML, não só a superfície |
| 🧩 **Simplificação de sistemas complexos** | Transforma problemas de negócio caóticos em modelos tratáveis |
| 📖 **Didática e comunicação** | Explica resultados complexos para leigos — habilidade rara em times de dados |

## 📊 Projetos de Ciência de Dados

> Portfólio prático desenvolvido durante os estudos.
> Cada projeto parte de um problema real de negócio e chega até o deploy em produção.

---

### 🏘️ [Pipeline ETL & EDA — Inteligência Imobiliária (Airbnb)](https://github.com/Kotige/SEU_REPO)
**Problema:** Identificar os melhores bairros para investimento em aluguel temporário a partir de dados públicos.

Pipeline de coleta via scraping → limpeza estatística de outliers (IQR) → análise exploratória visual que revelou os 3 bairros com melhor custo-benefício para novos investidores.

`Python` `Pandas` `BeautifulSoup` `Seaborn` `ETL` `EDA`

---

### 📉 [Previsor de Cancelamento de Clientes — Telecom Churn](https://github.com/Kotige/SEU_REPO)
**Problema:** Antecipar quais clientes vão cancelar o contrato antes de eles saírem.

Classificador XGBoost com tuning bayesiano (Optuna), tratamento de classes desbalanceadas e **94% de ROC-AUC** — com Feature Importance que revelou os fatores críticos de evasão.

`Python` `XGBoost` `Scikit-Learn` `Optuna` `Machine Learning` `SMOTE`

---

### 👥 [Segmentação de Clientes — RFM + K-Means](https://github.com/Kotige/SEU_REPO)
**Problema:** Dividir a base de clientes em grupos acionáveis para campanhas de marketing direto.

Engenharia de features RFM (Recência, Frequência, Monetário) + clustering K-Means com PCA para visualização 2D, gerando personas com recomendações de ação para o time de marketing.

`Python` `Scikit-Learn` `K-Means` `PCA` `Plotly` `Pandas`

---

### 🚀 [API de Predição em Produção — FastAPI + Docker + CI/CD](https://github.com/Kotige/SEU_REPO)
**Problema:** Disponibilizar um modelo de ML como serviço consumível por qualquer sistema.

Microsserviço de inferência com FastAPI, conteinerizado em Docker, com rastreamento de experimentos no MLflow, testes com pytest e deploy automático via GitHub Actions.

`FastAPI` `Docker` `MLflow` `pytest` `GitHub Actions` `MLOps`

---

### 🤖 [Assistente de Pesquisa com IA Generativa — RAG + LangChain](https://github.com/Kotige/SEU_REPO)
**Problema:** Permitir busca semântica inteligente em manuais técnicos extensos.

Pipeline RAG com LangChain + ChromaDB que ingere PDFs, gera embeddings e responde perguntas em linguagem natural com base nos documentos, com arquitetura documentada no padrão C4 Model.

`Python` `LangChain` `ChromaDB` `FastAPI` `RAG` `LLM` `C4 Model`
